# Podcast Player

iPhone の Safari から使う自分用ポッドキャストプレイヤー。
ビルド不要の静的 Web アプリ（vanilla JS）で、GitHub Pages でホスティングする。

## 機能

- **番組検索・お気に入り登録** — iTunes Search API で番組名検索し、タップで登録。
  エピソード一覧も iTunes Lookup API から取得（最新200件）し、RSS から文字起こし情報等を補完
- **再生** — 再生/停止、15秒スキップ、30秒巻き戻し、シークバー、再生位置の自動保存・復元。
  再生中は全画面プレイヤー（上半分＝スクロール可能な概要、下半分＝操作）を表示し、
  一時停止・未再生時は画面下部のミニバー（タイトル＋再生ボタン）に自動で切り替わる。
  操作ボタンの下のアクション行に、閉じる「∨」（左下隅・再生を続けたまま格納）、
  倍速ボタン（タップで 1.0x/1.1x/1.2x をサイクル・番組ごとに保存/同期）、
  スリープタイマー（🌙・タップで20分カウントダウン開始、残り分を表示、再タップで解除）を配置。
  ページを閉じて開き直しても最後のエピソードがプレイヤーに復元され、中断位置から再開できる。
- **AI 質問チャット** — エピソード内容をコンテキストに Gemini へ自由に質問できる
  チャット画面。エピソード詳細の「AI に質問」ボタン、または再生画面の吹き出しボタンから開く。
  文字起こしがあれば全文を、なければ概要と生成済み AI 分析をコンテキストに使用。
  エピソードにない内容は Google 検索グラウンディングで調べ、箇条書きで簡潔に回答
  （「エピソード外の情報です」と明示）。会話は複数ターン対応（セッション中は履歴を保持）。
  入力欄は複数行の質問でも文頭が隠れないよう自動で高さが伸びる（一定以上は内部スクロール）
- **タイムスタンプジャンプ** — エピソード概要内・AI 回答内の `3:12` `1:02:33` 形式の
  時刻表記を自動でリンク化し、タップでその位置から再生
- **URL リンク化** — エピソード概要・AI 回答内の `https://...` を自動でタップ可能な
  リンクにし、新しいタブで開く
- **AirPods 対応** — ダブルタップ（次のトラック）で15秒スキップ、
  トリプルタップ（前のトラック）で30秒巻き戻し（Media Session API 経由）
- **番組ごとの再生範囲設定** — 冒頭◯秒をスキップして開始／終わり◯秒手前で自動終了
  （挨拶やエンディング曲を飛ばす用途。番組詳細ページの「再生・表示の設定」を開いて
  5秒刻みに設定・端末間同期対応。この設定パネルは通常は折りたたまれている）
- **エピソード検索・分類** — 番組詳細ページでタイトル・概要の部分一致絞り込み。
  タブで All（全件）/ Fresh（未再生）/ Playing（再生途中・「途中 ◯分」バッジ付き）に分類表示。
  新しい順／古い順の並び替えにも対応（全タブ共通）
- **概要表示** — 番組の説明とエピソードごとの概要を表示
- **AI 分析・クイズ** — Gemini API（gemini-2.5-flash）でエピソードの重要ポイント（箇条書き）＋
  立てるべき本質的な問い3つ（答え付き）＋理解度クイズ4択5問を生成。
  クイズは事実の暗記ではなく、核心的な主張の理解や話全体の構造・論理展開を問う形式。
  番組ごとに「再生開始時に自動生成」を ON にできる（番組詳細ページの設定・デフォルト OFF・
  生成済み/生成中のエピソードはスキップ）。
  「新着エピソードを自動で AI 分析」を ON にした番組は、アプリ起動時（同期完了後）に
  最新エピソードをチェックし、未分析なら自動で生成する（デフォルト OFF・端末間同期対応）
  - RSS に文字起こし（Podcasting 2.0 `<podcast:transcript>`）があればテキストから生成
  - なければエピソード音声を Gemini Files API にアップロードして音声から直接生成
- **iPhone 対応** — ホーム画面追加（PWA manifest）、ロック画面の再生操作（Media Session API）

## 使い方

1. デプロイした URL を iPhone の Safari で開く（ホーム画面に追加推奨）
2. ホーム画面左上の歯車アイコンから設定を開き、Gemini API キーを保存
   （[Google AI Studio](https://aistudio.google.com/apikey) で取得）
3. ホーム画面右上の虫眼鏡アイコンから番組名を検索して登録
4. 番組 → エピソードを開いて再生、「分析とクイズを生成」で AI 機能を使用

データ（お気に入り・API キー・生成結果・再生位置）はすべて端末の localStorage に保存される。
サーバーには何も送信されない（外部 API との通信のみ）。

## ローカルで動かす

```sh
npx serve .        # または python3 -m http.server 8000
```

## GitHub Pages へのデプロイ

初回のみ（`gh` CLI 認証済みであること）:

```sh
./deploy.sh
```

以降の更新は `git push` するだけで反映される。

## 複数端末での同期（Mac / iPhone）

お気に入り・プロキシ URL・生成済み要約/クイズ・再生位置・倍速設定を、
自分の GitHub アカウントの**非公開 Gist** 経由で端末間同期できる。

1. [gist スコープのトークンを作成](https://github.com/settings/tokens/new?scopes=gist&description=podcast-player-sync)
   （スコープは gist のみでよい）
2. 各端末の「設定」（ホーム画面左上の歯車）→「端末間の同期（GitHub）」に
   同じトークンを貼って「保存して同期」

同期は起動時・変更時（数秒後にまとめて）・アプリのフォアグラウンド復帰時・
再生の一時停止時に自動実行される。「今すぐ同期」で手動実行も可能。
競合時はセクション単位で新しい方が優先され、要約/クイズと再生位置はキー単位でマージされる。

**Gemini API キー・GitHub トークンは同期対象外**（各端末で個別に入力する）。
非公開 Gist も GitHub のシークレットスキャン対象になり得るためで、
実際にキーを含めていた際に Google から「leaked」判定されて無効化された経緯がある。
proxyUrl のみ同期されるが、同期で「削除」は伝播しない（非空値の反映のみ）ため、
片端末の操作で他端末の設定が消えることはない。

### 設定が消える場合（iOS の localStorage 自動削除）

iOS Safari は「ホーム画面に追加」していないサイトの localStorage を約7日で
自動削除することがある（ITP）。これにより API キー・トークン・プロキシ URL が
消えることがあるため、アプリは起動時に永続ストレージ（`navigator.storage.persist()`）を
要求する。許可されなかった場合は設定画面に注意書きを表示する。
**確実に保持するには Safari で「ホーム画面に追加」して起動すること。**

## 自前 CORS プロキシの設置（音声取得が 403 で失敗する場合）

Spotify/Anchor 系など多くの音声ホストはブラウザからの直接取得（CORS）を許可しておらず、
公開プロキシも大きい音声ファイルを拒否するため、「音声の取得に失敗しました (HTTP 403)」に
なることがある。その場合は Cloudflare Workers（無料枠で十分）に `cf-proxy/` の
プロキシを設置する。

```sh
cd cf-proxy
npx wrangler login    # 初回のみ。ブラウザで Cloudflare にログイン（アカウントは無料で作成可）
npx wrangler deploy
```

表示された URL（例 `https://podcast-proxy.xxx.workers.dev`）を使い、アプリの
「設定」（ホーム画面左上の歯車）の「自前 CORS プロキシ URL」に次の形式で登録する：

```
https://podcast-proxy.xxx.workers.dev/?url=
```

第三者による無断利用を防ぎたい場合はトークン認証を有効にする：

```sh
npx wrangler secret put AUTH_TOKEN   # 任意の文字列を設定
```

その場合の登録 URL は `https://podcast-proxy.xxx.workers.dev/?token=<設定した値>&url=`。

登録すると RSS・文字起こし・音声のすべての取得がこのプロキシを最優先で使うようになる。

## 制約

- エピソード一覧は iTunes Lookup API（CORS 対応）から直接取得するため安定して動くが、
  RSS 由来の情報（文字起こしタグ・番組説明）と音声からの AI 生成は公開 CORS プロキシ
  （allorigins / corsproxy.io）に依存する。不安定な場合は `js/net.js` の `PROXIES` を
  自前のプロキシ（Cloudflare Workers 等）に差し替える
- Gemini API キーはブラウザの localStorage に保存される。共用端末では使わないこと
- 音声からの生成はファイルサイズ・長さに応じて時間と API コストがかかる
